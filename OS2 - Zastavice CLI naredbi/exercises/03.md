## Zadaci za vježbu: OS1 i OS2 (27. 3. 2026.)

Zadatke možete raditi samostalno ili u paru. Rješenja učitavate na GitHub kroz Git CLI, a zatim učitavate poveznicu do repozitorija na Merlin.

Evidentirajte CLI naredbe koje koristite u rješavanju zadataka u pomoćnu tekstualnu datoteku - nazovite ju `commands.txt`.

<hr>

1. Izradite novi GitHub repozitorij naziva `os2-vjezba-node` i klonirajte ga na svoje računalo.

- repozitorij mora biti javan i mora imati zadanu `README.md` datoteku.
- **Ovo mora napraviti samo jedan član, ako radite u paru**.

<br>

2. Klonirajte repozitorij na svoje računalo koristeći Git CLI. Repozitorij klonirajte u _home_ direktorij.

<br>

3. Koristeći isključivo CLI, izradite novu datoteku naziva `index.js` sa sadržajem:

```javascript
console.log("Hello, <ime_kolege>!"); // ili vaše ime, ako radite sami
```

<br>

4. Dodajte datoteku `index.js` u _staging area_, zatim napravite commit s porukom `"Dodana index.js datoteka"`. **Ovo mora napraviti samo jedan član, ako radite u paru**.

- provjerite status repozitorija i povijest commitova (_logove_).

<br>

5. Učitajte promjene na GitHub koristeći odgovarajuću Git naredbu te provjerite da se datoteka `index.js` pojavila.

- ako imate poteškoća s učitavanjem promjena (npr. autentifikacija), preuzmite GitHub CLI alat na stranici https://cli.github.com/ i slijedite upute za autentifikaciju koristeći naredbu: `gh auth login`.
- status autentifikacije provjerite naredbom `gh auth status`.
- drugi član tima isto mora odraditi autentifikaciju na svom računalu

<br>

6. Dodajte kolaboratora (ako radite u paru) na repozitorij kroz GitHub GUI.

- odaberite repozitorij, idite na `"Settings"` > `"Collaborators"` > `"Manage access"` i dodajte GitHub korisničko ime/email kolege/ice. "Pozivnica" dolazi na email adresu.

<br>

7. **Nakon prihvaćanja pozivnice, drugi član tima** mora klonirati repozitorij na svoje računalo (ako radite sami, klonirajte ponovo repozitorij na drugoj putanji), napraviti promjenu u `index.js` datoteci i napraviti commit s porukom `"Promijenjena index.js datoteka"`.

- **drugi član tima učitava promjene na GitHub**. Provjerite jesu li obje promjene (od oba člana) vidljive u povijesti commitova (_logovima_).

<br>

8. Instalirajte Node.js JavaScript runtime na svoje računalo (ako već nemate). **Ovo obavezno moraju imati oba člana tima.**

- provjerite instalaciju pokretanjem naredbe `node -v` u terminalu. Ako nema verzije, Node.js možete preuzeti sa https://nodejs.org/ u obliku _installera_.

<br>

9. `npm` (_Node Package Manager_) je alat koji dolazi s Node.js-om i koristi se za upravljanje paketima (bibliotekama) i skriptama u JavaScript projektima.

- provjerite instalaciju pokretanjem naredbe `npm -v` u terminalu.

<br>

10. Unutar vašeg repozitorija, pokrenite naredbu `npm init` da biste inicijalizirali novi Node.js projekt.

- slijedite upute i popunite potrebne informacije (naziv paketa izjednačite s nazivom repozitorija, verziju ostavite na `1.0.0`, a _entry point_ postavite na `index.js`), za autore upišite imena i prezimena oba člana tima, a ostale opcije ostavite na zadanim vrijednostima.

<br>

11. Provjerite sadržaj trenutnog direktorija i datoteke `package.json` (naredba `cat <putanja>`).

- datoteka `package.json` sadrži informacije o vašem Node projektu, uključujući naziv, verziju, ovisnosti i skripte. Ova datoteka je ključna za upravljanje Node projektom.

<br>

12. Naredbom `node <putanja>` pokrenite `index.js` JavaScript datoteku i provjerite ispravnost ispisa u terminalu.

- dodajte `package.json` u _staging area_, napravite commit s porukom `"Dodana package.json datoteka"` i učitajte promjene na GitHub.

<br>

13. **Drugi član tima** mora preuzeti sinkronizirati promjene na svoje računalo te instalirati ovisnosti pokretanjem naredbe `npm install` **unutar direktorija projekta**.

- nikako nemojte ponovno inicijalizirati Node.js projekt budući da to radi novu `package.json` datoteku.
- nakon instalacije, pokrenite `index.js` datoteku i provjerite ispravnost ispisa u terminalu.

<br>

14. Koristeći GUI, **drugi član tima** uređuje `index.js` (možete kroz GUI) tako da ispisuje njegovo ime i ime vašeg kolege sto puta naizmjenično.

- napravite commit sa porukom `"Promijenjena index.js datoteka na GitHub-u"` i provjerite da su promjene vidljive u povijesti commitova (_logovima_).
- **Prvi član tima** mora sinkronizirati promjene na svoje računalo te pokrenuti `index.js` datoteku i provjeriti ispravnost ispisa u terminalu.
- Nakon toga, **prvi član tima** mora promijeniti stanje Git projekta na prethodni _commit_ i pokrenuti `index.js` datoteku da vidi razliku u ispisu.

15. Učitajte datoteku `commands.txt` na GitHub i provjerite da su sve naredbe koje ste koristili u rješavanju zadataka vidljive.
